Build Your First App

üìñ 30 min read ¬∑ üü° Intermediate ¬∑ üõ†Ô∏è Hands-on

Follow along step-by-step to build a complete ecology data dashboard. We'll use the Palmer Penguins dataset and vibecoding techniques to go from zero to deployed app.

What We're Building

An interactive penguin biodiversity explorer with:

  • Summary metrics (total count, species, islands)
  • Scatter plots colored by species
  • Distribution histograms
  • Sidebar filters
  • A data table

Step 1: Initial Prompt

Open your AI assistant (Claude, ChatGPT, etc.) and send this prompt:

Your First Prompt

I want to build a Streamlit web app called "Penguin Biodiversity Explorer" that explores the Palmer Penguins dataset. Load the data from this URL: https://raw.githubusercontent.com/allisonhorst/palmerpenguins/main/inst/extdata/penguins.csv Start simple: just load the data, show a title, and display the dataframe. Use st.set_page_config with wide layout and a penguin emoji as the icon.

The AI will generate something like this:

app.py ‚Äî v1 
import streamlit as st
import pandas as pd

st.set_page_config(page_title="Penguin Explorer", page_icon="üêß", layout="wide")

st.title("üêß Penguin Biodiversity Explorer")

@st.cache_data
def load_data():
    url = "https://raw.githubusercontent.com/allisonhorst/palmerpenguins/main/inst/extdata/penguins.csv"
    return pd.read_csv(url)

df = load_data()
st.dataframe(df)

‚úÖ Checkpoint: Test it!

Save the code as app.py and run streamlit run app.py. You should see a table of penguin data. If not, paste the error message back to the AI.

Step 2: Add Summary Metrics

Second Prompt

Great, that works! Now add three metrics at the top using st.columns and st.metric: total number of penguins, number of species, and number of islands. Also add a brief description below the title explaining the dataset.

Step 3: Add Visualizations

Third Prompt

Now add two plotly charts side by side using columns: 1. A scatter plot of bill_length_mm vs bill_depth_mm, colored by species 2. A scatter plot of flipper_length_mm vs body_mass_g, colored by species Add proper axis labels with units. Make both charts use the same color scheme.

Step 4: Add Filters

Fourth Prompt

Add a sidebar with filters: - Multi-select for species (default: all selected) - Multi-select for island (default: all selected) - Dropdown for sex (options: All, Male, Female) All charts and metrics should update when the filters change. Drop rows with missing values in the filtered data.

Step 5: Add More Analysis

Fifth Prompt

Add tabs to organize the content: "Overview", "Distributions", and "Data". In "Overview": keep the metrics and scatter plots. In "Distributions": add histograms for each numeric measurement (bill length, bill depth, flipper length, body mass), colored by species, arranged in a 2x2 grid. In "Data": show the filtered dataframe with a download button to export as CSV.

Step 6: Polish

Sixth Prompt

Final polish: - Add an "About" section at the bottom with the data source citation - Improve the color scheme (use a nature-inspired palette) - Add a box plot comparing body mass across species in the Overview tab - Make sure all charts have clear titles - Add the count of filtered vs total penguins in the sidebar

Step 7: Prepare for Deployment

Deployment Prep

Generate a requirements.txt file for this app listing all the Python packages needed. Also create a README.md file for the GitHub repository with a brief description, features list, and instructions to run locally.

üí° Total prompts: 7

In about 7 back-and-forth exchanges, you've built a complete, polished, interactive data dashboard. That's vibecoding. Each prompt built on the previous result, adding complexity incrementally.

Common Issues & Fixes

"ModuleNotFoundError: No module named 'plotly'"

Run: pip install plotly

"KeyError: 'column_name'"

The column name in your code doesn't match the actual CSV. Tell the AI: "I'm getting a KeyError for 'column_name'. The actual columns in my CSV are: [paste column names]"

"The chart is empty"

Usually means your filters are too restrictive. Check that your default filter values include data. Tell the AI: "The chart shows no data. I think the filters might be excluding everything by default."

"The app is slow"

Tell the AI: "The app is slow to load. Can you add @st.cache_data to the data loading function?"
