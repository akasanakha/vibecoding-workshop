Troubleshooting

üîß Reference ¬∑ üü¢ All levels

Errors happen. That's normal. The key vibecoding skill for debugging: copy the exact error message and paste it to the AI with context about what you were trying to do.

The Universal Fix

Debug Prompt Template

I'm getting this error when I [describe what you were doing]: [paste the FULL error message] Here's my current code: [paste your code] What's wrong and how do I fix it?

üí° This works 90% of the time

AI assistants are extremely good at reading error messages and suggesting fixes. The key is giving them the complete error and your code.

Common Errors

ModuleNotFoundError: No module named 'X'

What it means: A Python package isn't installed.

Fix:

pip install X

Common packages you might need: streamlit, pandas, plotly, numpy, folium, streamlit-folium

KeyError: 'column_name'

What it means: Your code references a column that doesn't exist in the dataframe.

Fix: Check your actual column names:

# Add this to your app temporarily to see column names:
st.write(df.columns.tolist())

Then tell the AI: "The column is called 'Species' not 'species'" (capitalization matters!).

FileNotFoundError: No such file or directory

What it means: The app can't find your data file.

Fix:

  • Use relative paths: "data.csv" not "/home/user/data.csv"
  • Make sure the file is in the same directory as your app.py
  • Better yet: load from a URL instead of a local file

"StreamlitAPIException" or white screen

What it means: Something in your Streamlit code is incorrect.

Fix: Check the terminal where you ran streamlit run app.py ‚Äî the full error will be there. Paste it to the AI.

Charts show no data / empty charts

What it means: Usually your filters are excluding all data, or there are NaN values.

Fix:

# Add after filtering to check:
st.write(f"Filtered data has {len(filtered_df)} rows")
st.write(filtered_df.head())

# Drop NaN values before plotting:
plot_df = filtered_df.dropna(subset=["column_x", "column_y"])

"TypeError: unhashable type" with st.cache_data

What it means: Streamlit's caching is having trouble with your function.

Fix: Tell the AI: "I'm getting a cache error. Can you modify the cached function to avoid unhashable types?"

App is extremely slow

Possible causes:

  • Loading large datasets without caching ‚Äî add @st.cache_data
  • Too many data points on a map ‚Äî sample your data: df.sample(1000)
  • Complex computations on every interaction ‚Äî cache intermediate results

Deployment-Specific Issues

App works locally but crashes on Streamlit Cloud

  • Check requirements.txt ‚Äî are ALL packages listed?
  • Check file paths ‚Äî no absolute paths!
  • Check Streamlit Cloud logs: Manage app ‚Üí Logs

"Your app is having trouble loading"

  • The app might have an error on startup ‚Äî check logs
  • Memory limit exceeded ‚Äî reduce data size or optimize
  • Package installation failed ‚Äî check requirements.txt for typos

Getting Help During the Workshop

  1. First: Copy the error and ask your AI assistant
  2. Second: Try the relevant fix from this page
  3. Third: Ask a workshop instructor
  4. Last resort: Start fresh from the checkpoint template
