The Art of Prompting

üìñ 15 min read ¬∑ üü¢ Beginner

The quality of your prompts determines the quality of what AI builds for you. This guide teaches you how to communicate effectively with AI coding assistants.

The Five Golden Rules

1. Be Specific

Vague prompts produce vague results. Compare:

‚ùå Too Vague

"Make a chart of my data"

‚úÖ Specific

"Make a bar chart showing the count of species observations per habitat type. Use plotly, with habitat on the x-axis and count on the y-axis. Color each bar by habitat type."

2. Go Step by Step

Don't ask for the entire app at once. Build incrementally:

  1. First: "Create a Streamlit app that loads my CSV file and displays it as a table"
  2. Then: "Add a scatter plot of column_x vs column_y"
  3. Then: "Add a sidebar filter for the species column"
  4. Then: "Add a second tab with a map view"

3. Show Your Data Structure

The AI doesn't know your data unless you tell it. Always describe your columns:

Good Prompt

"My CSV file has these columns: site_id (string), date (YYYY-MM-DD), latitude (float), longitude (float), species_name (string), abundance (integer), habitat_type (categorical: forest/wetland/grassland). Load this file and create a map showing each site, with point size proportional to total abundance."

4. Iterate, Don't Start Over

When something isn't right, describe what to change instead of starting fresh:

Iteration Prompt

"The chart looks good, but: (1) make the title larger, (2) change the color scheme to viridis, (3) add axis labels with units, and (4) remove the legend since there's only one variable."

5. Ask Why

If the AI does something unexpected, ask it to explain:

Understanding Prompt

"Why did you use st.cache_data here? What does the @st.cache_data decorator do?"

Prompt Templates for Ecology Apps

Starting a New Project

Template 
I want to build a Streamlit web app that:
- Loads a CSV dataset about [TOPIC]
- The dataset has columns: [LIST YOUR COLUMNS]
- Shows [SPECIFIC VISUALIZATION]
- Lets users [SPECIFIC INTERACTION]

Use Python with streamlit and plotly.
Let's start with loading and displaying the data.

Adding a Chart

Template 
Add a [CHART TYPE] that shows [WHAT TO PLOT].
- X-axis: [column name] ([units])
- Y-axis: [column name] ([units])
- Color by: [column name]
- Add a title: "[YOUR TITLE]"
Use plotly for interactivity.

Adding a Map

Template 
Add an interactive map showing [WHAT].
- Latitude column: [name]
- Longitude column: [name]
- Color points by: [column]
- Size points by: [column or "uniform"]
- Add popups showing: [what info on click]
- Center the map on: [location or "auto"]

Adding Filters

Template 
Add a sidebar with these filters:
- Dropdown to filter by [COLUMN] (allow multiple selections)
- Date range picker for [DATE_COLUMN]
- Slider for [NUMERIC_COLUMN] range
All visualizations should update when filters change.

Fixing Errors

Template 
I'm getting this error when I run the app:

[PASTE THE FULL ERROR MESSAGE]

The error occurs in [DESCRIBE WHERE/WHEN].
Here's the relevant code section:

[PASTE THE CODE]

What's wrong and how do I fix it?

Improving the UI

Template 
Improve the layout of this app:
- Add a title and description at the top
- Organize the content into [N] tabs: [tab names]
- Use a clean, professional color scheme
- Add units and labels to all charts
- Make it responsive on mobile
- Add an "About" section explaining the data source

Common Prompting Mistakes

‚ùå Being too ambitious at once

"Build me a complete biodiversity monitoring platform with user authentication, real-time data ingestion, machine learning predictions, and a mobile app."

‚ùå Not providing context

"Add a chart." (What chart? What data? What columns?)

‚ùå Accepting the first result

Always test what the AI generates. If something doesn't look right, tell it what's wrong. The best results come from 3-5 iterations.

‚ùå Not reading error messages

When you get an error, copy the ENTIRE error message and paste it to the AI. Don't paraphrase ‚Äî the exact text matters.

Pro Tips

  • Start each session with context: "I'm building a Streamlit app for [purpose]. Here's my current code: [paste]. I want to add [feature]."
  • Ask for alternatives: "Show me 3 different ways to visualize this species abundance data"
  • Request explanations: "Add comments to this code explaining what each section does"
  • Use screenshots: Some AI tools accept images. Screenshot your current app and say "How do I improve this layout?"
  • Save your best prompts: When a prompt works well, save it for reuse
