What is Vibecoding?

üìñ 10 min read ¬∑ üü¢ Beginner

Vibecoding is building software by describing what you want in plain English, and letting AI write the code for you.

The term was coined by Andrej Karpathy, co-founder of OpenAI, in February 2025:

Karpathy's Definition

"There's a new kind of coding I call 'vibe coding', where you fully give in to the vibes, embrace exponentials, and forget that the code even exists. I just see stuff, say stuff, run stuff, and copy paste stuff, and it mostly works."

Why This Matters for Scientists

As ecologists, you have deep domain expertise. You understand species interactions, nutrient cycling, spatial patterns, and ecosystem dynamics. But turning that knowledge into interactive tools traditionally required years of programming training.

Vibecoding changes the equation: your domain expertise IS the most valuable skill. The AI handles the syntax; you handle the science.

What you know that AI doesn't:

  • Which variables matter in your dataset
  • What visualization would be most informative
  • How to interpret the patterns
  • What questions your audience cares about
  • What would actually be useful for your research

What AI knows that you might not:

  • Python/JavaScript syntax
  • Web framework patterns
  • CSS styling
  • Deployment configurations
  • Library APIs

How It Works in Practice

Here's a real example of vibecoding an ecology app:

You Say

"Build me a Streamlit app that loads the Palmer Penguins CSV dataset. Show a scatter plot of bill length vs bill depth, colored by species. Add a sidebar where I can filter by island and sex."

The AI generates a complete, working Python script. You run it. It works (mostly). Then you iterate:

You Say

"Great, but make the chart bigger, add a trendline for each species, and put summary statistics in a table below the chart."

Each iteration takes seconds. Within minutes, you have a polished interactive dashboard that would have taken days to build from scratch.
